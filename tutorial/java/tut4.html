Arrays!

A convient way of making multiple variables for the price of one!

So we talked just a bit about arrays. But we haven't seen or used them yet?

Well we are now! But let's first define what an arrays is.

An Array is a variable that stores other variables. Or put simply, more than one value concurrently.

Arrays also like standard variables have only one set type.

This means a array that is an int may not store types that are Strings. Unlike standard variables however

An array must STRICTLY use it's OWN TYPE , unless you typecast it but that's a different lesson.

The syntax for declaring / defining an array is not too complicated and is relatively easy to grasp.

*datatype* *variablename* [] = new *datatype*[*sizeofarray*]

Here is an example

float number[] = new number[512];

Now your liking asking, how do I assign values to arrays?

Arrays assign values to their indices, which are what contains a variable.

Arrays are infact memory address pointing to other memory address that it owns(You make new memory with the new keyword)

Since making arrays can consume more memory than making a singular variable. Don't make an array bigger than you need.

Now to access the indices of an array.

*variablename*[*indexNumber*] = *valueThatMatchesDataType*

Example

number[256] = 125.1918f;

One thing to note

The first index is ZERO, not ONE as you would believe

this is because that arrays require one byte reserved to be the terminating byte. Which is the escape code \0

Most modern and advanced languages automatically add it on, but if you do not have one and use a lower level language

like assembly, your variable will access memory that doesn't belong to it and output absolute GARBAGE (with a free program crash!).

back to the assignment of array values. Well if you have a big array isn't it going to be redundant?

Yes... Yes it is, this why we use our friend, the FOR LOOP

Down below is the common implementation of a for loop to fill an array

for ( int i = 0;; i < *size*; i++ )

array[i] = *value*

see? Much less annoying than doing a hundred lines of array[*number*] = *value* eh?

Arrays don't have to be specified types. WHA???

Yep that's right. Arrays are not just limited to the standard variable types in Java ( same for other languages )

Once we get to Object Orientation you'll see examples of this in action, just keep this as a hint for the future
